# 遅延ラベリングと創発的概念形成

この文書は、動画・画像・イベントなどの観測から、先に特徴や構造を形成し、後から人間が名前を付与する方式について整理した研究仮説である。

現時点では確定実装仕様ではなく、RISAおよびKnowledge Serverの将来設計へ反映する候補として扱う。

## 基本アイデア

人間が最初からすべてのデータへラベルを付けるのではなく、AIが観測データから共通構造を抽出し、類似するものを自動的に集める。

```text
動画・画像・イベント
→ 特徴抽出
→ 類似構造のクラスタ化
→ 概念候補の生成
→ 必要な候補だけ人間が命名
→ 概念・関係・証拠として保存
```

従来型の教師あり学習では、入力データに対して人間が先に正解ラベルを付ける。ここで扱う方式では、構造形成を先に行い、ラベル付けを後段へ遅延させる。

## 観測から構造へ

AIはまず、以下のような観測特徴を利用して、データ間の類似性を形成する。

- 動き
- 速度
- 軌道
- 接触
- 音
- 色
- 形
- 状態変化
- 時間的な前後関係

例えば、机、壁、人を対象とした複数の「叩く」動画から、腕の振り、急加速、衝突、反発などの共通パターンを抽出し、名前のないクラスタとして保持する。

```text
Cluster 18
→ 人間が「叩く」と命名
```

人間はすべての動画へ個別ラベルを付けるのではなく、代表的なクラスタや概念候補に名前を付ける。

## 未知概念の発見

クラスタ化によって、人間が事前に定義していなかった共通パターンを発見できる可能性がある。

```text
Cluster 231
・人が困っている
・少し後ずさりする
```

このようなクラスタは、既存のラベルに無理に割り当てず、未知概念候補として保存する。人間が確認した後、新しい概念名、既存概念との関係、適用文脈を追加する。

## 概念形成の順序

人間の概念形成に近い順序として、以下を仮説とする。

```text
特徴形成
→ 概念形成
→ 言語付与
```

初期段階では概念は名前を持たなくてもよい。名前、説明、別名、言語コードは後から付与できる属性として設計する。

## RISAとの関係

RISAは知識を重みだけでなく、関係構造として保持・更新することを目指すため、名前のない特徴クラスタをノードとして扱える。

```text
観測
→ 特徴ノード
→ クラスタノード
→ 概念ノード
→ 言語ラベル
```

クラスタや概念は、以下の情報を持つ候補としてKnowledge Serverへ保存できる。

- 観測データへの参照
- 共通特徴
- 類似度
- 概念候補の階層
- 既存概念との関係
- 人間によるラベル
- 信頼度
- 反例
- 生成元と生成バージョン
- 検証状態

## Knowledge Serverでの扱い

初期段階では、自然言語や動画を直接自律処理する機能を実装せず、将来拡張可能なデータ構造を優先する。

候補となる保存単位:

```text
観測データ
特徴表現
クラスタ
概念候補
概念ラベル
概念間関係
根拠イベント
人間レビュー
```

確定概念と未検証の概念候補を、`verification_state`や候補種別で分離する。人間が命名した事実と、AIが発見したクラスタを同じ確度の知識として扱わない。

## 期待される効果

- 大量の個別ラベル作成を削減できる可能性
- 人間が事前に定義していない概念を発見できる可能性
- 動作、物体、関係、因果などを同じ構造で扱える可能性
- 後から複数の言語ラベルを付与できる
- LLM、SARA、RISAで同じ概念資産を再利用できる

## 注意点

特徴クラスタが意味のある概念になるとは限らない。単なる撮影条件、背景、話者、データセット固有の偏りをクラスタ化する可能性がある。

したがって、以下を評価機構に含める。

- クラスタの再現性
- データセット変更に対する安定性
- 背景や撮影条件への過適合
- 人間ラベルとの整合性
- 予測や検索への有用性
- 反例を含めた境界の明確さ
- 概念形成にかかる計算コスト

## 将来の研究課題

- クラスタ分裂・融合・休眠
- 概念階層の自動形成
- 動画・音声・テキスト間の概念対応
- 名前のない概念の検索と可視化
- 人間の少数ラベルによる概念同定
- 概念候補からイベント予測への利用
- RISAの関係更新とKnowledge Serverの履歴管理の分離
